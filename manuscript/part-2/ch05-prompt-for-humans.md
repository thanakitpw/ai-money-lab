# บทที่ 5
## Prompt ฉบับคนธรรมดา — ไม่ต้องเรียน Prompt Engineering

---

## 5.1 บทนี้สอนอะไร

สมมติว่าสองคนนั่งข้าง ๆ กัน ใช้ ChatGPT ตัวเดียวกัน เวอร์ชันเดียวกัน

คนแรกพิมพ์

> "ช่วยเขียนแคปชันขายกาแฟหน่อย"

คนที่สองพิมพ์

> "คุณเป็นคนเขียนแคปชันขายของบน Facebook ช่วยเขียนแคปชันสำหรับกาแฟเย็นราคา 65 บาท ขายในคาเฟ่ที่เชียงใหม่ ลูกค้าเป็นนักเรียน ม.มช. อายุ 19-22 tone เป็นกันเองนิด ๆ ขอ 3 แบบ ความยาวไม่เกิน 60 คำ จบด้วยคำชวนให้แชร์"

คน ๆ เดียวกัน AI เดียวกัน แต่ผลลัพธ์ออกมาต่างกันฟ้ากับเหว

คนแรกได้แคปชันแบบทั่วไปที่ใช้ไม่ได้จริง

คนที่สองได้ 3 แคปชันที่ copy ไปโพสต์ได้เลย

ความต่างไม่ได้อยู่ที่ AI ความต่างอยู่ที่ prompt

บทนี้สอนสูตรเดียว 4 ข้อ ที่ทำให้คุณเขียน prompt ได้แบบคนที่สอง ไม่ต้องไปเรียนคอร์ส "Prompt Engineering" ไม่ต้องจำ syntax พิเศษ ใช้ได้กับ AI ทุกตัว

---

## 5.2 ทำไมไม่ต้องเรียน Prompt Engineering

ก่อนเข้าสูตร อยากพูดถึงคำนี้ก่อน

"Prompt Engineering" เป็นคำที่คนสอนคอร์สชอบใช้ ให้ฟังดูยาก ให้คุณรู้สึกว่าต้องจ่าย 3,000-10,000 เพื่อเรียน เทคนิคลับ เช่น chain-of-thought, few-shot learning, temperature tuning

ความจริงที่ไม่มีใครบอก

คนใช้งานจริงส่วนใหญ่ไม่ได้เรียน Prompt Engineering มาก่อน พวกเขาแค่เรียนสูตรพื้นฐานที่ทำให้ได้ผลลัพธ์ดี

เขาใช้ prompt แบบภาษาคน บอก AI ว่าต้องการอะไร ให้บริบท ให้ตัวอย่าง ลองดู ปรับ ทำซ้ำ

Prompt Engineering แบบที่สอนในคอร์สแพง ๆ มีประโยชน์ในบริบทเฉพาะ เช่นสร้าง agent, ทำ LLM-based product, fine-tune model สำหรับงานเฉพาะ สำหรับคนที่อยากหาเงินเสริมด้วย AI คุณไม่ต้องไปถึงตรงนั้น

สิ่งที่คุณต้องทำเป็น
- บอก AI อย่างชัดว่าต้องการอะไร
- ให้บริบทพอที่ AI ทำงานได้ตรง
- อ่านผลลัพธ์เป็น รู้ว่าต้องปรับตรงไหน

แค่นี้พอ

## 5.3 สูตร 4 ข้อ — บทบาท + งาน + บริบท + สไตล์

สูตรที่ใช้ได้กับทุก prompt ไม่ว่าจะเขียนแคปชัน แปลเอกสาร สรุปงาน ทำ Excel formula หรือ plan trip

4 ข้อนี้ต้องมีทุกครั้ง ถ้าขาดข้อไหน ผลลัพธ์จะง่อน

### ข้อ 1: บทบาท (Role)

บอก AI ว่า "ตอนนี้คุณเป็นใคร"

AI เวลาไม่ได้ถูกสั่งบทบาท จะตอบแบบทั่วไป แต่พอบอกบทบาท AI ปรับวิธีคิดให้ตรงกับอาชีพนั้น

ตัวอย่าง

- "คุณเป็นนักเขียนแคปชัน Facebook สำหรับธุรกิจเล็กในไทย"
- "คุณเป็น copywriter โฆษณา Facebook Ads ที่ทำให้คนปิดการขายได้ดี"
- "คุณเป็นครูสอนภาษาไทยให้นักเรียนชั้นประถม"
- "คุณเป็นที่ปรึกษาด้านภาษีสำหรับฟรีแลนซ์"

บทบาทไม่ต้องโก้หรูห่างไกล แค่ระบุชัดว่าให้ AI คิดจากมุมไหน

### ข้อ 2: งาน (Task)

บอก AI ว่า "ต้องทำอะไร"

ตรงนี้หลายคนคิดว่าใส่ไปแล้ว แต่ใส่ไม่ชัด

ตัวอย่างเปรียบเทียบ

แบบไม่ชัด "ช่วยเขียนแคปชันหน่อย"
แบบชัด "ช่วยเขียนแคปชัน Facebook 3 variation สำหรับโพสต์ขายสินค้า"

แบบไม่ชัด "ช่วยสรุปเอกสารนี้"
แบบชัด "ช่วยสรุปเอกสารนี้เป็น bullet 7 ข้อ พร้อมตัวเลขสำคัญในแต่ละข้อ"

งานต้องระบุ output ที่ต้องการ (แคปชัน 3 แบบ / bullet 7 ข้อ) ไม่ใช่แค่ action

### ข้อ 3: บริบท (Context)

ข้อที่คนลืมบ่อยที่สุด แต่สำคัญที่สุด

AI ไม่รู้อะไรเกี่ยวกับสถานการณ์ของคุณ ธุรกิจของคุณ ลูกค้าของคุณ ถ้าคุณไม่บอก AI จะเดา และการเดาของ AI มักจะเป็น "ค่ากลาง" ของโลก ซึ่งไม่ใช่สิ่งที่คุณต้องการ

บริบทที่ควรใส่

- สินค้า / บริการ / หัวข้อเฉพาะ (ชื่อ ราคา จุดเด่น)
- กลุ่มเป้าหมาย (เพศ อายุ ความสนใจ)
- สถานที่ / ช่องทาง (Facebook / LINE / Instagram)
- ข้อจำกัด (ความยาว งบ deadline)

ตัวอย่าง

```
บริบท
สินค้า กาแฟเย็น cold brew ราคา 75 บาท
ร้าน คาเฟ่เล็กในย่านอารีย์ กรุงเทพ
กลุ่มเป้าหมาย พนักงานออฟฟิศ 25-35 ปี ที่แวะซื้อระหว่างไปทำงาน
ช่องทาง Facebook page ของร้าน
ข้อจำกัด ความยาวไม่เกิน 80 คำ ห้ามใช้คำว่า "อร่อย" "คุ้ม"
```

พอใส่บริบทครบ AI ทำงานได้ตรงมาก

### ข้อ 4: สไตล์ (Style)

ข้อสุดท้าย ต้องบอก AI ว่า "เสียงที่ต้องการเป็นยังไง"

AI แบบค่าเริ่มต้นจะเขียนเรียบ ๆ ไม่มีเอกลักษณ์ ถ้าคุณไม่บอกสไตล์

ตัวอย่าง

- "tone เป็นกันเอง เหมือนเพื่อนแนะนำ"
- "tone ทางการ แบบหนังสือราชการ"
- "tone อารมณ์ขัน เล่นคำ แต่ไม่ยัด pun"
- "เขียนเหมือนคนไทยอายุ 40+ พูด ไม่ใช่คนสิบกว่าพูด"

### Prompt ตัวอย่างเต็มรูป

รวม 4 ข้อเข้าด้วยกัน

```
[บทบาท] คุณเป็นนักเขียนแคปชัน Facebook สำหรับธุรกิจเล็กในไทย

[งาน] ช่วยเขียนแคปชัน 3 variation สำหรับโพสต์ขายสินค้าต่อไปนี้

[บริบท]
สินค้า กาแฟเย็น cold brew ราคา 75 บาท
ร้าน คาเฟ่เล็กในย่านอารีย์ กรุงเทพ
กลุ่มเป้าหมาย พนักงานออฟฟิศ 25-35 ปี
ช่องทาง Facebook page
ข้อจำกัด ไม่เกิน 80 คำ ห้ามใช้คำว่า "อร่อย" "คุ้ม"

[สไตล์] tone เป็นกันเอง เหมือนเพื่อนแนะนำ ไม่ใช่ sales ยัด
```

Copy ไปใช้ได้เลย แค่เปลี่ยน [บริบท] ให้ตรงกับงานของคุณ

## 5.4 ตัวอย่างเทียบชัด — Prompt ทั่วไป vs Prompt 4 ข้อ

### ตัวอย่างที่ 1: เขียนแคปชัน

**Prompt ทั่วไป**
```
เขียนแคปชันขายเสื้อยืด
```

**ผลลัพธ์ที่มักได้**
> "เสื้อยืดคุณภาพดี เนื้อผ้านุ่ม ใส่สบาย เหมาะสำหรับทุกโอกาส สั่งซื้อได้เลยวันนี้!"

generic ใช้ไม่ได้จริง

**Prompt 4 ข้อ**
```
คุณเป็นนักเขียนแคปชัน Facebook

ช่วยเขียนแคปชัน 2 variation สำหรับเสื้อยืด

สินค้า เสื้อยืด oversize ผ้า cotton 100% ราคา 390 บาท
ลูกค้า ผู้หญิงอายุ 22-30 ที่ work from home
ช่องทาง Facebook page
ข้อจำกัด ไม่เกิน 60 คำ ห้ามใช้คำว่า "คุณภาพดี" "ใส่สบาย"

สไตล์ เป็นกันเอง ใช้ภาษาพูดแบบ Gen Y จบด้วย CTA ชวนให้ทัก
```

**ผลลัพธ์ที่มักได้**
> "วันหยุดนี้ไม่อยากใส่ชุดที่กดตัว ลอง oversize cotton 100% ของเรา ใส่ทำงานที่บ้าน นอนเล่นเกมเย็น ใส่ไปซื้อของเสาร์อาทิตย์ได้หมด 390 บาท ทักเราเลยค่ะ"

มีบริบท มีอารมณ์ มี CTA ชัด ใช้ได้ทันที

### ตัวอย่างที่ 2: สรุปบทความ

**Prompt ทั่วไป**
```
สรุปบทความนี้ให้หน่อย
[paste บทความยาว]
```

**ผลลัพธ์ที่มักได้** สรุปย่อ 1 ย่อหน้า ไม่เน้นตรงไหน อ่านแล้วไม่รู้จะเอาไปทำอะไร

**Prompt 4 ข้อ**
```
คุณเป็นนักวิเคราะห์ที่สรุปงานให้ผู้บริหารอ่าน 5 นาทีเข้าใจ

ช่วยสรุปบทความต่อไปนี้ใน format
1. TL;DR 1 ประโยค
2. ประเด็นหลัก 5 ข้อเป็น bullet
3. ตัวเลขสำคัญ 3 ตัวพร้อมบริบท
4. สิ่งที่ผู้อ่านควรทำถัดไป 2 ข้อ

บริบท ผู้อ่านเป็นเจ้าของร้านอาหารเล็ก ไม่มีเวลาอ่านบทความยาว
สไตล์ ภาษาไทยเรียบง่าย ไม่ใช้ศัพท์ธุรกิจ

บทความ
[paste บทความ]
```

**ผลลัพธ์ที่มักได้** สรุปมี structure ชัด เอาไปใช้ต่อได้ทันที

### ตัวอย่างที่ 3: เขียนอีเมล

**Prompt ทั่วไป**
```
เขียนอีเมลปฏิเสธงานลูกค้าที่ราคาไม่ถึง
```

**ผลลัพธ์ที่มักได้**
> "เรียน คุณลูกค้า ผมขอปฏิเสธงานนี้เนื่องจากราคาไม่ตรงกับเรทของเรา ขอบคุณสำหรับความสนใจ"

ห้วน หักหลัง ลูกค้าไม่กลับมาแน่

**Prompt 4 ข้อ**
```
คุณเป็นฟรีแลนซ์เขียน content ที่รักษาความสัมพันธ์ลูกค้าเก่ง

ช่วยเขียนอีเมลปฏิเสธงาน

บริบท
ลูกค้าขอให้เขียน blog 5 บทความ 1,000 คำในงบ 1,500 บาทรวม (เรทตลาด 3,000 บาท/บทความ)
คุณอยากปฏิเสธแต่ไม่อยากปิดประตู
ลูกค้าเป็นคนดี อาจมีงานอื่นในอนาคต

เงื่อนไข
- ปฏิเสธอย่างสุภาพ อธิบายเรทที่เหมาะ
- เสนอทางเลือก (ถ้ามีงบเท่านี้ ทำอะไรได้บ้าง)
- เปิดช่องให้กลับมาคุยถ้างบขยับ

สไตล์ เป็นกันเองแต่ professional ความยาวไม่เกิน 150 คำ
```

**ผลลัพธ์ที่มักได้** อีเมลที่รักษาความสัมพันธ์ เปิดโอกาสลูกค้ากลับมา

## 5.5 เทคนิค "สั่งซ้ำ" — พอผลลัพธ์ยังไม่ใช่

เขียน prompt ดีแล้ว แต่ผลลัพธ์ออกมายังไม่ถูกใจ อย่ารีบเขียน prompt ใหม่ทั้งก้อน

ใช้การ "สั่งซ้ำ" ภายใน conversation เดิม AI จำ context ได้ ปรับทีละจุดได้

### 5 ประโยคปรับผลลัพธ์ที่ใช้บ่อย

**หนึ่ง "สั้นลง ประมาณครึ่งหนึ่ง"**

ได้ผลลัพธ์ยาวไป

**สอง "ขอแบบที่ tone กันเองกว่านี้ ใช้ภาษาพูด"**

ได้ผลลัพธ์ทางการเกิน

**สาม "ขออีก 2 variation ที่ต่างจากเดิม"**

ได้ผลลัพธ์แค่แบบเดียว อยากเห็นตัวเลือก

**สี่ "อันนี้ใกล้แล้ว ปรับตรง [ส่วนที่ต้องแก้] แค่นั้น"**

ได้ผลลัพธ์ใช้ได้ 80% แต่มีจุดเล็ก ๆ ที่ผิด

**ห้า "ลองเขียนให้เหมือน [คน/แบรนด์ที่คุณชอบ] มากขึ้น"**

ได้ผลลัพธ์สไตล์ไม่ตรง

### ตัวอย่าง conversation flow

```
คุณ ช่วยเขียนแคปชันขาย cold brew...
AI [generate แคปชัน]
คุณ อันนี้ใกล้แล้ว แต่ tone เป็นทางการเกิน ขอกันเองกว่านี้ ใช้ภาษาพูด
AI [generate ใหม่]
คุณ ดีขึ้น ขอให้สั้นลงเหลือ 50 คำ จบด้วย CTA ชวนให้แชร์
AI [final version]
```

3-4 รอบเสร็จ ใช้เวลา 10 นาทีรวม ได้งานคุณภาพ

ถ้าเขียน prompt ใหม่ทุกครั้งที่ไม่ถูกใจ จะเสียเวลามากกว่า เพราะ AI ลืม context เก่าต้องสร้างใหม่

## 5.6 5 ข้อผิดพลาดที่คนใหม่ชอบทำ

จาก pattern ที่เห็นบ่อยในคนเริ่มใช้ AI

### หนึ่ง Prompt สั้นเกิน

> "เขียนแคปชันขายของ"

AI ไม่รู้จักของ ไม่รู้จักคน ไม่รู้จักช่องทาง ได้ผลลัพธ์ทั่วไปที่ใช้ไม่ได้

แก้ ใช้สูตร 4 ข้อเสมอ

### สอง คาดหวังผลลัพธ์ perfect ครั้งแรก

คนใหม่ถ้าอ่านผลลัพธ์แรกแล้วไม่ถูกใจ มักสรุปว่า "AI ใช้ไม่ได้" แล้วเลิก

ความจริง ผลลัพธ์แรกของ AI คือ 70% ของผลลัพธ์สุดท้าย ต้องสั่งซ้ำ 2-3 รอบ ถึงได้ 95%

แก้ ตั้งใจว่าจะคุยกับ AI อย่างน้อย 3 รอบ ก่อนได้ output final

### สาม ถาม AI ในสิ่งที่ AI ไม่รู้

"ยอดขายของฉันเดือนนี้เป็นยังไง" → AI ไม่รู้
"กลุ่มเป้าหมายจริงของฉันคือใคร" → AI เดา ได้ค่ากลาง
"ข่าวล่าสุดเรื่อง..." → AI ที่ไม่เชื่อม web ตอบไม่ได้

แก้ ถาม AI ในสิ่งที่เป็นความรู้ทั่วไป + ใส่ข้อมูลเฉพาะของคุณเข้าไป ไม่ต้องคาดหวังให้ AI รู้เอง

### สี่ ไม่เช็คข้อมูลก่อนเอาไปใช้

AI "hallucinate" บ่อย สร้างตัวเลข สถิติ อ้างอิง ที่ดูเหมือนจริงแต่ไม่จริง

ถ้า AI บอกว่า "จากการวิจัยของ Harvard ปี 2565..." ต้องไป Google ก่อนว่ามีจริงมั้ย ส่วนใหญ่ไม่มี

แก้ ตัวเลข สถิติ ชื่อบุคคล ชื่อสถานที่ ต้อง verify ก่อนเผยแพร่ทุกครั้ง

### ห้า ใช้ AI สำหรับทุกอย่าง

AI ไม่ใช่เครื่องมือ universal ถ้าคำถามเป็น "ควรเลือก supplier ไหน" สำหรับธุรกิจคุณ ไม่ใช่คำถาม AI ควรตอบ เพราะต้องเข้าใจบริบทเฉพาะของธุรกิจคุณ

แก้ AI เหมาะกับงานที่มี pattern ชัด (เขียน แปล สรุป สร้าง idea) ไม่เหมาะกับการตัดสินใจที่ต้องเข้าใจ context ส่วนตัวลึก

บทที่ 4 พูดเรื่องนี้ละเอียดกว่า (AI ทำอะไรได้ ทำอะไรไม่ได้)

## 5.7 Prompt Template สำเร็จรูป 3 แบบที่ใช้บ่อย

Copy ไปเก็บไว้ เปลี่ยน [ช่องว่าง] ตามงาน

### Template 1: สำหรับงานเขียน

```
คุณเป็น [บทบาท เช่น นักเขียนแคปชัน / copywriter / content writer]

ช่วยเขียน [ประเภทงาน เช่น แคปชัน Facebook / ad copy / blog บทความ]

บริบท
- หัวข้อ/สินค้า [ระบุ]
- กลุ่มเป้าหมาย [ระบุ]
- ช่องทาง [ระบุ]

เงื่อนไข
- ความยาว [ระบุจำนวนคำ]
- ขอ [จำนวน] variation
- ห้ามใช้คำว่า [รายการคำที่ไม่ต้องการ]

สไตล์ [ระบุ tone]
```

### Template 2: สำหรับงานแปล / สรุป

```
คุณเป็น [นักแปล / นักวิเคราะห์]

ช่วย [แปล / สรุป] เอกสารต่อไปนี้

บริบท
- เอกสารใช้สำหรับ [จุดประสงค์]
- ผู้อ่าน [กลุ่มเป้าหมาย]

เงื่อนไข
- Format ที่ต้องการ [bullet / paragraph / ตาราง]
- ความยาว [ระบุ]
- รักษา [ชื่อเฉพาะ / ตัวเลข / ศัพท์เทคนิค] ตามต้นฉบับ

สไตล์ [ระบุ tone]

[paste เอกสาร]
```

### Template 3: สำหรับงานวิเคราะห์ / ให้คำแนะนำ

note "วิเคราะห์" ที่หมายถึงที่นี่คือ structured summary เปรียบเทียบทางเลือก pros/cons แบบที่ AI ทำได้ ไม่ใช่ deep data analysis ที่ต้องตัวเลขเชิงลึก ซึ่ง บทที่ 4 เตือนว่า AI ทำพลาดบ่อย การตัดสินใจสุดท้ายยังเป็นของคุณ AI แค่ช่วยจัดโครงความคิด

```
คุณเป็น [ที่ปรึกษา / ผู้เชี่ยวชาญ] ด้าน [สาขา]

ช่วย [วิเคราะห์ / แนะนำ / เปรียบเทียบ] ต่อไปนี้

บริบท
- สถานการณ์ [อธิบาย]
- สิ่งที่ลองแล้ว [ถ้ามี]
- ข้อจำกัด [งบ / เวลา / ทักษะ]

เงื่อนไข
- ขอ [จำนวน] ทางเลือก
- แต่ละทางต้องมี pros / cons
- แนะนำทางที่ดีที่สุดพร้อมเหตุผล

สไตล์ ตรงไปตรงมา ไม่อ้อม ใช้ภาษาคน
```

## 5.8 สรุปบทที่ 5 + Action

### สรุป

Prompt ที่ดีไม่ต้องเรียน Prompt Engineering

ต้องมีแค่ 4 ข้อ บทบาท + งาน + บริบท + สไตล์

ผลลัพธ์แรกของ AI คือ 70% ของ output final สั่งซ้ำ 2-3 รอบเพื่อเก็บอีก 25-30% ที่เหลือ

หลีกเลี่ยง 5 ข้อผิดพลาด prompt สั้นเกิน / คาดหวัง perfect ครั้งแรก / ถามสิ่งที่ AI ไม่รู้ / ไม่เช็คข้อมูล / ใช้ AI กับทุกอย่าง

Template 3 แบบในหัวข้อ 5.7 คือฐานที่ใช้ได้กับ 80% ของงาน copy เก็บไว้ใช้

### Action

**วันนี้ ตอนนี้**

เปิด ChatGPT หรือ Claude ใช้ Template 1 เขียน prompt ขอให้ AI เขียน "แคปชัน Facebook 3 แบบ สำหรับ [ของที่คุณนึกออก] ลูกค้า [ใครก็ได้ที่คุณคิดถึง]"

อ่านผลลัพธ์ แล้ว "สั่งซ้ำ" อย่างน้อย 2 ครั้ง

ใช้เวลา 15 นาที

ถ้าผลลัพธ์ดีขึ้นจากรอบแรก แปลว่าคุณเข้าใจบทนี้แล้ว
